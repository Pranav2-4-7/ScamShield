TELECOM LAYER
─────────────
telecom/ScamShieldInCallService.kt    – Custom dialer core
                                         • Receives all incoming/outgoing calls
                                         • Detects unknown callers via ContactsHelper
                                         • Auto-switches to speakerphone for unknowns
                                         • Starts/stops ScamDetectionService
telecom/ScamCallCallback.kt           – Handles call state transitions (ringing →
                                         active → disconnected)
telecom/ScamShieldConnectionService.kt – ConnectionService for dialer registration
telecom/ScamShieldConnection.kt        – Individual call connection implementation

AUDIO PIPELINE
──────────────
audio/AudioCaptureManager.kt    – Mic-based PCM audio capture
                                   • AudioSource.VOICE_RECOGNITION (noise-optimised)
                                   • 16 kHz mono, 16-bit PCM
                                   • 500ms streaming chunks
                                   • Noise gate + amplitude normalisation
                                   • Ephemeral: buffer overwritten each iteration

audio/SpeechToTextEngine.kt     – On-device speech recognition
                                   • Android SpeechRecognizer API
                                   • Prefers offline mode (EXTRA_PREFER_OFFLINE)
                                   • en-IN locale (handles Indian English accents)
                                   • Continuous mode with auto-restart on errors
                                   • Partial results for low-latency detection

audio/SlidingWindowBuffer.kt    – Rolling transcript context window
                                   • Max 150 words / 30 segments
                                   • Oldest context auto-evicted
                                   • Provides context to ML without storing full log

ML DETECTION
────────────
ml/ScamDetectionEngine.kt       – Dual-mode scam detection engine
                                   • PRIMARY: TFLite Bidirectional LSTM model
                                     (loads from assets/scam_detection.tflite)
                                   • FALLBACK: Rule-based regex engine (10 patterns)
                                   • Neural Network API (NNAPI) hardware acceleration
                                   • Combined scoring: max(ML_score, rule_score)
                                   • Risk levels: Low/Medium/High/Critical

SCAM PATTERNS DETECTED (10 categories):
  1. OTP_REQUEST             – OTP, verification code, PIN requests
  2. BANK_IMPERSONATION      – SBI/HDFC/ICICI/AXIS impersonation, KYC threats
  3. GOVERNMENT_IMPERSONATION – CBI, digital arrest, TRAI, narcotics, income tax
  4. URGENCY_TACTIC          – "act now", "within 24 hours", "don't disconnect"
  5. PRIZE_SCAM              – KBC, lottery, lucky draw
  6. TECH_SUPPORT            – AnyDesk, TeamViewer, Microsoft/Apple impersonation
  7. REFUND_SCAM             – Fake refund requiring bank details
  8. PERSONAL_INFO_REQUEST   – Aadhaar, PAN, CVV, passwords
  9. KYC_SCAM                – KYC expiry threats, video KYC scams
  10. REMOTE_ACCESS          – Install apps, screen share requests

SERVICES
────────
service/ScamDetectionService.kt  – Foreground orchestration service
                                    • Starts with ACTION_START on call answer
                                    • Runs pipeline: Audio → STT → Window → ML
                                    • Emits risk scores and alerts via StateFlow
                                    • Triggers vibration on HIGH/CRITICAL alerts
                                    • Updates persistent notification with risk level
                                    • Stops with ACTION_STOP on call end
                                    • Clears all buffers after stopping

UTILITIES
─────────
utils/PhoneStateHolder.kt   – Ephemeral in-memory state (Kotlin object)
                               • StateFlow streams for UI reactivity
                               • clearAll() called on every call end
                               • NEVER persisted to disk

utils/ContactsHelper.kt     – ContactsContract lookup for known/unknown check

PRIVACY DESIGN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• No audio is recorded. PCM buffers are overwritten after each 500ms chunk.
• No transcripts are stored. SlidingWindowBuffer is in-memory only.
• No call logs written by ScamShield.
• No network calls. All ML inference runs on-device.
• No cloud APIs. SpeechRecognizer uses EXTRA_PREFER_OFFLINE.
• Backup excluded via backup_rules.xml and data_extraction_rules.xml.
• PhoneStateHolder.clearAll() called on every call end.
• PostCallActivity snapshots summary before clearAll(), then displays it
  ephemerally in UI only (never written to disk).
